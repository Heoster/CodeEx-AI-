## ğŸ¯ Key Design Decisions

### 1. Service Layer Pattern
- Centralized auth logic in `FirebaseAuthService`
- Reusable across components and API routes
- Easy to test and maintain

### 2. API Routes for Security
- Server-side validation
- Token verification
- Rate limiting capability

### 3. React Hook for Convenience
- Optional abstraction layer
- Built-in loading states
- Simplified error handling

### 4. Separate Pages
- `/reset-password` - Public access
- `/account-settings` - Authenticated access
- Clear separation of concerns

### 5. Component Composition
- Reusable form components
- Consistent UI/UX
- Easy to integrate anywhere
